    *   *Verification:* Ensure `package.json` and `requirements.txt` exist.
*   **Step 1.2: Styling Foundation**
    *   Configure Tailwind in the frontend.
    *   Inject the specific corporate color variables (e.g., `--bg: #0b1e3f; --teal: #0d8a8a`) into the root CSS.
    *   *Verification:* Ensure CSS variables are accessible globally in the React app.

### Phase 2: High-Fidelity UI (Mock Data Mode)
*   **Step 2.1: Layout & Core KPI Dashboard**
    *   Translate the static HTML prototype into a responsive 3-column React masonry grid.
    *   Create `Header.jsx` and `KpiRow.jsx`. Use static JSON to populate the numbers temporarily.
    *   *Verification:* React app renders without console errors.
*   **Step 2.2: Complex Widgets**
    *   Build `ManageEngine.jsx` (Station Heatmap, SLA charts using `chart.js`).
    *   Build `M365Widgets.jsx` (Inbox triage list, Planner tabs).
    *   *Verification:* All UI components match the visual density of the prototype.

### Phase 3: Backend Database & Core API
*   **Step 3.1: FastAPI Boilerplate**
    *   Initialize `main.py` with standard REST routes (`/api/v1/manageengine`, `/api/v1/m365`).
    *   Configure strict CORS policies to allow requests from the Teams client environment.
    *   *Verification:* Uvicorn server starts successfully on port 8000.
*   **Step 3.2: Enterprise SQL Architecture**
    *   Implement `database.py` using `sqlalchemy.ext.asyncio`.
    *   Configure robust connection pooling (`pool_size=10, max_overflow=20`) to prevent DB exhaustion.
    *   *Verification:* Create dummy SQL queries using parameterized inputs (`text("SELECT * FROM tickets WHERE user_email = :email")`).

### Phase 4: Zero-Friction Authentication (The Teams OBO Flow)
*   **Step 4.1: Teams SDK Integration**
    *   Install `@microsoft/teams-js` in React.
    *   Create `useTeamsAuth.js` hook. **Crucial:** Implement an environment toggle. If `VITE_ENV=local`, return a mock JWT. Otherwise, call `authentication.getAuthToken()`.
    *   *Verification:* The app successfully generates a token (or mock token) on load.
*   **Step 4.2: Microsoft Graph OBO Security**
    *   In FastAPI, write a middleware dependency to validate the incoming Teams JWT.
    *   Implement the MSAL Confidential Client to exchange the Teams Token for a Microsoft Graph Access Token (On-Behalf-Of flow).
    *   *Verification:* The backend can successfully intercept a token and extract the user's `preferred_username` (email).

### Phase 5: Live Wiring & System Resilience
*   **Step 5.1: Replace Mocks with Live APIs**
    *   Update React widgets to fetch from `http://localhost:8000/api/v1/...` passing the Teams token in the `Authorization: Bearer` header.
    *   Ensure FastAPI uses `asyncio.gather()` to fetch Mail, Calendar, and Planner concurrently, not sequentially.
    *   *Verification:* UI populates based on API responses.
*   **Step 5.2: Fault Tolerance (40-Year Architect Standard)**
    *   **Frontend:** Wrap `ManageEngine.jsx` and `M365Widgets.jsx` in React Error Boundaries. Add pulsing CSS skeleton loaders during fetch states.
    *   **Backend:** Implement a 1-minute caching layer (in-memory or Redis) for the SQL queries to prevent DB locks. Implement `try/except` for HTTP 429 Graph API throttling. If Graph fails, serve cached data.
    *   *Verification:* Manually breaking the SQL connection string should gracefully show an error in the ManageEngine widget while leaving the M365 email widget fully functional.
